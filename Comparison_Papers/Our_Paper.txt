Abstract
Deformable linear object (DLO) perception in monocular images requires recovering ordered centerlines under
severe occlusion and self-occlusion, where segmentationfirst pipelines often fail to preserve curve order. We present
an assignment-aware transformer framework for ordered
centerline prediction based on a LETR/DETR-style architecture [2, 1]. The method combines curvature-driven annotation/group construction, global direction selection, and
LSAP-based correspondence. We study four supervision
variants spanning direct dense-curve Chamfer supervision
(v2), group-aware deterministic matching with and without
extra matches (v8/v9), and stochastic in-group representative sampling via a discrete normal distribution (v10). For
compute-constrained model selection, we use a staged tournament protocol with standardized dense-curve evaluation
and validation-threshold selection. This framework provides
a reproducible basis for analyzing how matcher design affects robustness and ordered centerline quality in occlusionheavy DLO scenes.
1. Introduction
The core task in this work is predicting ordered centerline
points of linear objects in images. This is our formulation of
the deformable linear object (DLO) perception problem in
monocular imagery. A primary challenge is severe occlusion:
linear objects can intersect other objects or self-intersect
repeatedly, so centerline segments disappear and reappear
along the image. Under these conditions, pure segmentation formulations are often unsuitable because they do not
directly enforce a consistent ordered centerline parameterization through occluded regions. In this project, we evaluate several model variants that share a common backbonetransformer architecture but differ in how prediction-target
correspondences are constructed and supervised.
This paper presents an implementation-grounded methods
and evaluation framework for comparing assignment and
supervision strategies for centerline prediction. Methods
and experiment protocol sections are detailed, and results
sections provide structured tables for quantitative reporting.

2. Related Work
2.1. Deformable Linear Object Perception
Perception of deformable linear objects (DLOs) has been
studied across sutures, ropes, and cables, where the core challenge is to recover a stable centerline under large deformation, appearance ambiguity, and frequent occlusion. We emphasize recent literature (primarily 2020+): learning-based
pipelines that predict reliable keypoints before spline reconstruction [5], dataset/transferability benchmarks that expose domain-shift limits in 3D DLO understanding [7], and
topology-preserving multi-DLO tracking for highly entangled scenes [8]. Recent DLO-focused datasets and transferability studies also highlight that generalization across object
types (e.g., wiring vs. vessels) remains a central bottleneck
even for strong 3D methods [7]. For multi-object, highly
entangled settings, topology-preserving tracking methods
such as MultiDLO emphasize geodesic/topological consistency to maintain identities and local structure under occlusion [8]. However, segmentation-first approaches still require topology reconstruction and ordering post-processing,
which becomes brittle under repeated self-occlusions and
crossings. Broader recent surveys similarly emphasize that
perception errors under complex deformation directly degrade downstream manipulation and planning reliability [6].
The RMDLO open-source ecosystem further reflects this
trend toward integrated perception-tracking-manipulation
pipelines for wires and ropes [10].

2.2. Set Prediction and Assignment for Curvilinear
Targets
Set-prediction frameworks with bipartite assignment
(DETR-style) provide a natural way to supervise unordered
query outputs [1, 4]. For curvilinear prediction, the open
challenge is that geometric supervision depends not only on
point location, but also on consistent point ordering along
the curve parameter. This motivates assignment designs that
explicitly couple correspondence, directionality, and ordering variables. In parallel, foundation-model segmentation
approaches for DLOs (e.g., CLIPSeg+SAM adapters) report
strong mask quality on cable datasets, but still target instance
masks rather than ordered centerline parameterization [9].
2.3. Positioning of This Work
Our variants target the failure modes above by combining:
• explicit dist-based ordering with global direction selection,
• group-aware LSAP correspondence that is less sensitive
to local target noise than point-wise-only matching,
• controlled extra-match augmentation (v8/v10) to increase useful supervision under noisy labels,
• stochastic in-group representative sampling (v10) as
regularization against overfitting to a single deterministic representative.
In short, relative to pure segmentation-then-reconstruction
pipelines, this project pushes supervision directly into matching and ordered set prediction, which is better aligned with
centerline parameterization under occlusion-heavy DLO
scenes.

4. End-to-End Architecture
4.1. Backbone and Feature Interface
The model uses a ResNet backbone (default: ResNet-50)
with sine positional encodings. While the backbone can
expose intermediate maps from C2–C5, the downstream architecture consumes only selected levels. In the base singletransformer pipeline (used by v2/v8/v9/v10), one deep feature map (C5) is projected by a 1 × 1 convolution into transformer dimension d = 256 and used as the transformer input.
In the stacked two-stage pipeline, a second, higher-resolution
map (C4) is additionally consumed by the refinement transformer. Spatial masks from the nested-tensor input are propagated so attention ignores padded pixels.

4.3. Two-Stage Multi-Resolution Pipeline
The framework supports a two-stage training/inference
path for explicit multi-resolution fusion:
• Stage 1 (coarse/global): train the base transformer on
the deeper backbone feature map (C5).
• Stage 2 (refinement/higher-res): freeze the stage-1
stack, then add a second transformer that consumes a
higher-resolution feature map (C4) through a new 1×1
projection.
In stage 2, the final stage-1 decoder states are passed as
query tokens to the second transformer decoder. This integrates global ordering/context from stage 1 with finer local
geometry from the higher-resolution map, improving sensitivity to thin structures and local bends. An optional stage-3
fine-tuning phase keeps this stacked structure and changes
optimization/loss settings (e.g., focal classification).
4.4. Output Heads
Default heads in this project are:
• classification head: linear or residual-MLP depending
on variant,
• line head: residual-MLP to 2D coordinates,
• dist head: residual-MLP to 1D dist.
By default, line and dist outputs use sigmoid activation, i.e.,
pˆq = σ(fxy(hq)),
ˆdq = σ(fd(hq)).
4.5. Variant-Specific Architectural Notes
• v2: classification head is removed from learning path
(dummy logits emitted only for API compatibility).
• v8/v9/v10: architecture remains base-like; differences
are matcher/loss behavior.

6.1. Data and Splits
Dataset organization follows COCO-style image and annotation splits. Training and validation use predefined singlecurve annotation subsets. Default preprocessing resizes images to 500 × 500 before normalization and augmentation.

6.3. Training Setup
All compared variants are trained with the same staged
multi-phase training procedure. Training uses distributed
execution over up to 3 GPUs.
Stage schedule used in tournament round-2. Current
tournament defaults run three stages per variant:
• Stage 1: 120 epochs, LR drop at epoch 100.
• Stage 2: 120 epochs, LR drop at epoch 100 (checkpoint
handoff with frozen lower stack; this is the explicit
higher-resolution fusion stage).
• Stage 3: 40 epochs, LR drop at epoch 20 (fine-tuning;
focal classification loss enabled in v8/v9/v10).
6.4. Tournament Protocol
A tournament-style orchestration procedure provides tworound logic. In the current configuration, round 1 is disabled
by default and all variants proceed directly to round 2.
For each variant (and optionally each seed), the harness:
1. launches staged training,
2. evaluates the final stage checkpoint using a unified
curve metric,
3. stores per-run records in CSV,
4. ranks variants by mean score (lower is better),
5. generates summary plots (ranking bars, training-loss
curves, threshold statistics).
6.5. Evaluation Metric
Evaluation computes dense curve Chamfer in 3D [x, y, d]:
1. order predicted control points by predicted dist,
2. fit dense Catmull–Rom spline in XY,
3. linearly interpolate dist along curve arclength,
4. compute symmetric Chamfer distance to target [x, y, d]
points.
By default, Chamfer uses p = 1 (L1 metric).

6.7. Threshold Search for Classification Variants
For variants with prediction-level class probabilities, the
evaluator sweeps inclusion threshold t ∈ [0.4, 1.0] with step
0.02. For each threshold, it recomputes dense curve Chamfer over the validation split and selects the best threshold
per variant/checkpoint. This controls for threshold sensitivity when comparing to variants that do not use class-gated
selection.

7.2. Limitations
• LSAP cost is still based on nearest 3D point projection,
not full-sequence alignment.
• Dist ordering relies on non-differentiable sorting for
control-point ordering.
• Tournament comparison is budget-limited and may not
reflect full-convergence ranking.

9. Conclusion
This draft provides a concrete, implementation-faithful
manuscript scaffold for the current centerline-variant study.
The key methodological axis is assignment design: from
global curve-level supervision (v2) to group-aware LSAP
with deterministic or stochastic representatives (v8/v9/v10),
and optional extra matching for robustness to noisy line inputs. Filling the provided results templates after tournament
completion will yield a near-ready technical paper draft.
